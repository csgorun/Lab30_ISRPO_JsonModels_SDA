## ⚙️ Атрибуты и настройки JSON

| Атрибут / настройка | Что делает | Пример результата |
|---------------------|-----------|------------------|
| `[JsonPropertyName("name")]` | Задаёт точное имя поля в JSON вместо имени свойства C# | `"name": "Человек-паук"` вместо `"Name": "Человек-паук"` |
| `[JsonIgnore]` | Полностью исключает поле из сериализации | Поле `InternalNotes` не попадает в ответ |
| `PropertyNamingPolicy = JsonNamingPolicy.CamelCase` | Автоматически преобразует все имена свойств в camelCase | `RealName` → `realName`, `PowerLevel` → `powerLevel` |
| `JsonStringEnumConverter` | Сериализует enum как строку, а не число | `"universe": "Marvel"` вместо `"universe": 0` |
| `WriteIndented = true` | Добавляет отступы и переносы для читаемости | Красивый JSON с форматированием (удобно для разработки) |

## 💡 Главные выводы

### 1. Что такое сериализация и десериализация?
> **Сериализация** — это процесс преобразования объекта C# в текстовый формат (JSON), который можно передать по сети или сохранить в файл.  
> **Десериализация** — обратный процесс: восстановление объекта C# из JSON-строки.  
>   
> *Пример*: Объект `Hero { Name = "Thor" }` → сериализация → `{"name":"Thor"}` → сеть → десериализация → новый объект `Hero`.

### 2. Зачем нужен `[JsonIgnore]`? Реальный пример
> Атрибут `[JsonIgnore]` исключает поле из JSON-ответа, что критически важно для:
> - **Безопасности**: не передавать клиенту пароли, хеши, токены доступа
> - **Инкапсуляции**: скрыть служебные данные (внутренние заметки, отладочную информацию)
> - **Оптимизации**: не отправлять лишние данные, уменьшая размер ответа
>   
> *Пример из проекта*: Поле `InternalNotes` содержит заметки разработчика ("Любимый герой редактора"). Клиенту эта информация не нужна — она скрыта атрибутом `[JsonIgnore]`.

### 3. Почему enum лучше сериализовывать строкой, а не числом?
> По умолчанию enum сериализуется как число (`Universe.Marvel` → `0`). Это проблема, потому что:
> - **Непонятно клиенту**: что значит `0`? Нужно искать документацию
> - **Хрупкость**: если изменить порядок значений в enum, числа сдвинутся и сломают совместимость
> - **Читаемость**: `"universe": "Marvel"` сразу понятно, `0` — нет
>   
> *Решение*: `JsonStringEnumConverter` преобразует enum в строку, делая API самодокументируемым и устойчивым к изменениям.

### 4. Что изменилось после настройки CamelCase?
> До настройки:
> ```json
> { "Id": 1, "Name": "Thor", "RealName": "Тор Одинсон", "Universe": 0 }
> ```
>   
> После настройки (`PropertyNamingPolicy.CamelCase` + `JsonStringEnumConverter`):
> ```json
> {
>   "id": 1,
>   "name": "Thor",
>   "realName": "Тор Одинсон",
>   "universe": "Marvel"
> }
> ```
>   
> **Преимущества**:
> - Соответствие стандартам JSON/JavaScript (camelCase — общепринятый стиль)
> - Удобство для фронтенд-разработчиков
> - Читаемость и предсказуемость ответов API
